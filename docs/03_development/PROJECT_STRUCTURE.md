Documentation Principles

Each document should have one clear responsibility.

Avoid duplicated content.

When architecture changes:

Update PROJECT_STRUCTURE.md first.

Then update the affected documents.

⸻

Long-term Goal

The project should grow in functionality without growing in confusion.

Documentation should make the architecture easier to understand, easier to maintain, and easier to evolve.
